# AntiGravity 懶人包 #00：一次安裝全部

> 版本：v2.0
>
> 更新日期：2026-07-17

## 這份懶人包會做什麼

本檔與根目錄 `SKILL.md` 是安裝派送入口，不會另行安裝成額外 Skill。它們會把五個功能 Skill 依序安裝到 Antigravity IDE 的全域 Skill 目錄，並逐項確認是否要執行設定：

1. `antigravity-notebooklm`
2. `antigravity-github`
3. `antigravity-firebase`
4. `antigravity-draw`
5. `antigravity-obsidian`

安裝目標固定為：

```text
~/.gemini/config/skills/<skill-name>/
```

Windows 對應：

```text
C:\Users\<使用者>\.gemini\config\skills\<skill-name>\
```

## 執行原則

- 先列出五項與目前安裝狀態。
- 每一項安裝或執行前都先詢問使用者，不整批無提示執行。
- 已安裝的 Skill 先檢查內容與版本，不直接覆蓋。
- 安裝 Skill 不等於立即執行其中的外部登入、建立專案、建立筆記或生圖。
- 遇到瀏覽器登入、API Key、全域套件安裝或外部寫入時暫停，交由使用者確認。

## 安裝方式

下載或 clone 本 repo 後，安裝全部五個功能 Skill：

```powershell
.\scripts\install-skills.ps1 -All
```

只安裝單一 Skill 時，必須使用 frontmatter 的完整名稱，例如：

```powershell
.\scripts\install-skills.ps1 -Skill antigravity-notebooklm
```

安裝器會逐項詢問；若目標已存在，預設不覆蓋，確認要更新時才加 `-Force`。只有已在外部流程取得完整授權的自動化情境，才使用 `-Confirm:$false`。無法執行安裝器時，才手動把對應 Skill 資料夾複製到 `~/.gemini/config/skills/`。

## 驗收

1. 確認五個 Skill 資料夾存在。
2. 每個資料夾都有合法的 `SKILL.md`。
3. `name` 與安裝清單一致。
4. 重新啟動 AntiGravity，確認能辨識 Skill。
5. 回報每一項為：已安裝／已存在／跳過／失敗。

## 復原

若只要移除某一項，刪除該 Skill 的全域資料夾即可；刪除前先列出完整路徑並取得使用者同意。移除 Skill 不會自動登出 GitHub、Google、Firebase 或刪除既有專案資料。
