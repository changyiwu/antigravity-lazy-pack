# AntiGravity 懶人包 #05：設定專案工作流程

> 版本：v2.0
>
> 更新日期：2026-07-17

## 固定規則

本懶人包統一使用專案根目錄的 `ANTIGRAVITY.md` 作為主要規則入口。開工、收工與新專案初始化都先讀取它，不改用其他規則檔取代。

全域 Skill 固定安裝到：

```text
~/.gemini/config/skills/<skill-name>/
```

## `ANTIGRAVITY.md` 應記錄什麼

- 專案名稱、用途與主要工作目錄。
- GitHub repo 與預設 branch。
- 對應的 Obsidian vault／專案駕駛艙。
- 固定工作規則、驗證指令與安全邊界。
- 不要存放每天的進度流水帳、token、密碼或個人匯出資料。

## 開工

使用者說「開工」時：

1. 讀取專案根目錄的 `ANTIGRAVITY.md`。
2. 依其中設定讀取 Obsidian 專案駕駛艙。
3. 執行 `git status --short --branch` 與最近 commit 檢查。
4. 回報目前狀態、未完成事項、風險與建議下一步。
5. 不自動 pull、commit 或 push。

## 收工

使用者說「收工」時：

1. 檢查本次變更與敏感資料。
2. 更新 Obsidian 專案駕駛艙的完成事項、下一步與踩坑。
3. 只有固定規則或路徑改變時才更新 `ANTIGRAVITY.md`。
4. 執行 `git status`、diff 與適合的專案驗證。
5. 只 stage 本次相關檔案，不使用無差別 `git add .`。
6. commit／push 前確認目標 branch、remote 與使用者授權。
7. 回報本機、Obsidian 與 GitHub 的同步結果。

## 新專案初始化

先確認：

- 專案名稱與用途。
- 工作資料夾。
- 是否建立 GitHub repo，以及公開／私有。
- 是否需要部署。
- Obsidian vault 與駕駛艙位置。

接著盤點並只補缺口：

- `ANTIGRAVITY.md`
- `README.md`
- `.gitignore`
- Git repo
- GitHub repo（使用者需要時）
- Obsidian 專案駕駛艙

既有檔案不可直接覆蓋；先顯示已存在／缺少清單。

## `ANTIGRAVITY.md` 範本

```markdown
# <專案名稱> - ANTIGRAVITY.md

## 專案入口

專案名稱：
專案用途：
主要工作目錄：
GitHub repo：
預設 branch：main

## Obsidian 對應筆記

Obsidian vault：
專案駕駛艙：

## 工作規則

- 回應使用繁體中文。
- `ANTIGRAVITY.md` 是主要規則入口。
- 使用 PowerShell 語法。
- 開工時讀本檔、駕駛艙與 Git 狀態。
- 收工時先驗證與檢查 diff，只提交相關檔案。

## 不要做

- 不要提交 API key、token、密碼或憑證。
- 不要提交 NotebookLM 個人清單。
- 不要自動納入無關變更。
```

## 驗收

- `ANTIGRAVITY.md` 存在且路徑正確。
- 規則、Git remote／branch 與實際專案一致。
- 駕駛艙可讀寫。
- 開工流程維持唯讀；收工與初始化的外部寫入都有明確授權。
